29 /**
30 * @defgroup globus_gram_protocol_framing Message Framing
31 * @ingroup globus_gram_protocol_functions
32 *
33 * The functions in this section take GRAM request (or query) and
34 * reply messages, and frame them with HTTP headers, so that they can be
35 * sent. These functions should be used when an application wants to control
36 * the way that the GRAM Protocol messages are sent, while still using
37 * the standard message formatting and framing routines. An alternative
38 * set of functions in the @ref globus_gram_protocol_io section of the manual
39 * combine message framing with callback-driven I/O.
40 */
